# Design Specs

Screen designs generated by Claude Code. Each file is a self-contained HTML mockup that can be opened directly in a browser.

## How to add a design from Claude Code

1. In Claude Code, ask for a screen design (e.g. *"Create an HTML mockup for the ProfileScreen"*)
2. Save the output to the correct subfolder:
   - **Mobile (Flutter)** → `specs/designs/mobile/<screen-name>.html`
   - **Web UI (React)** → `specs/designs/ui/<page-name>.html`
3. Use kebab-case filenames that match the screen/page name in the spec (e.g. `profile-screen.html`, `dashboard-page.html`)
4. Add a row to the index table below

## Referencing a design in a screen spec

Inside `specs/mobile/screens.md` or `specs/ui/components.md`, add a **Design** link to the relevant screen/page row:

```markdown
| `ProfileScreen` | `/profile` | Yes | ⏳ v2 | … | [design](../designs/mobile/profile-screen.html) |
```

## Conventions

- One HTML file per screen/page — keep each file self-contained (inline CSS/JS, no external deps)
- Use the project design tokens when prompting Claude:
  - Primary colour `#3B82F6`, Font `Inter`, Spacing base `4 px`
- Annotate interactive states (hover, focus, error) as separate sections within the same file
- Do **not** commit real user data or screenshots; mockups only
